# 🧭 運営方針 (Governance Policy)

このドキュメントは、Luminos-Studio-Dev がどのように意思決定を行い、メンバーの役割を定義するかを透明化するためのものです。

## 1. 組織の価値観

私たちの運営は、以下の価値観に基づいています。

* **透明性:** 可能な限り、すべての議論と意思決定プロセスを公開します。
* **公平性:** すべてのメンバー、貢献者に対し、公平で開かれた機会を提供します。
* **創造性:** 開発者が自由にアイデアを追求し、組織のビジョンに沿った新しい作品を生み出すことを奨励します。

## 2. メンバーの役割と責任

組織のメンバーは、権限と責任に基づき、以下のロールに分類されます。

| ロール | 主な責任 | 意思決定権 |
| :--- | :--- | :--- |
| **Owner** | 組織全体の設定、セキュリティ、財務の管理。最終決定権。 | Veto権を含む**全権** |
| **Core Member** | 主要プロジェクトの技術的指導、コードレビュー、PRマージの実行。 | 自身の専門分野に関する**技術的決定権** |
| **Contributor** | バグ修正、機能追加、ドキュメント作成など、特定のタスクへの貢献。 | **投票権** (コミュニティ意見として) |

## 3. 意思決定プロセス

### A. 一般的な技術的決定

1.  **議論:** 関連する Issue、またはDiscordで議論を開始します。
2.  **提案 (Proposal):** Core Memberが具体的な解決策を提案します。
3.  **合意形成:** Core Memberがレビューと議論を通じて合意形成を目指します。
4.  **決定:** 合意が得られた場合、Core Memberによって実装されます。

### B. 運営上の重要な決定

組織の名称変更、新たな規約の制定など、組織の根幹に関わる決定は、**OwnerとCore Member全員の合議**によって行われます。

* 議論はプライベートチャネルで行われる場合がありますが、結果と理由についてはコミュニティに公開されます。

### C. 新しい Core Member の選出

継続的に高品質な貢献（コード、レビュー、ドキュメント）を行い、組織の価値観を体現している Contributor は、Core Memberからの推薦と Owner の承認を経て昇格します。
