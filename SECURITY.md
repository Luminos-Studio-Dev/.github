# 🛡️ セキュリティポリシー (Security Policy) - Luminos-Studio-Dev

Luminos-Studio-Dev では、私たちのプロジェクトを安全に保つために、セキュリティ研究者やユーザーからの脆弱性報告を真摯に受け付けています。

## 🚨 1. 脆弱性の報告方法（How to Report a Vulnerability）

**いかなる場合も、Issueトラッカーや公開の場で脆弱性を報告しないでください。**

機密性の高いセキュリティ情報は、必ず以下のプライベートな方法で報告をお願いいたします。

### 🥇 推奨: GitHub Private Vulnerability Reporting (PVR)

GitHub の機能を利用した非公開の報告システムです。最も迅速かつ安全に対応可能です。

* リポジトリの「Security」タブから「**Report a vulnerability**」を選択し、フォームを送信してください。

### 🥈 代替手段: 専用メールアドレス

上記PVRが利用できない場合、以下の専用メールアドレスにご連絡ください。

* 📧 **メールアドレス:** `security@luminos-studio-dev.org` (仮)
* [もしあれば] PGPキーで暗号化を推奨します。 [PGP Public Key Link]

#### 報告に含めていただきたい情報

1.  **脆弱性の種類:** (例: XSS, SQL Injection, Logic Errorなど)
2.  **影響を受けるプロジェクトとバージョン:** (例: Web App v1.2.0)
3.  **再現手順:** 脆弱性を再現するための具体的な手順。
4.  **深刻度:** 報告者による評価（例: High, Medium）。
5.  **連絡先:** こちらから連絡を取るためのメールアドレス。

## ⏳ 2. 脆弱性への対応プロセス (Our Commitment)

報告を受け取った後、私たちは以下のステップで対応いたします。

1.  **受領確認:** 報告から **48時間以内** に、受領した旨の確認メールを送信します。
2.  **評価:** セキュリティチームが脆弱性の再現性と深刻度を評価します。
3.  **修正と調整:** 修正作業を行い、パッチリリースの日程を調整します。
4.  **公開:** 修正版がリリースされた後、脆弱性の詳細と**報告者への謝意**を公開します。

## 🟢 3. サポート対象バージョン (Supported Versions)

私たちは、以下のバージョンのプロジェクトに対してセキュリティアップデートを提供します。

* **最新の安定バージョン:** `main` ブランチに追従している最新リリース
* **それ以前のバージョン:** 原則として、セキュリティ修正は最新バージョンにのみ適用されます。

## 🏆 4. 謝意 (Acknowledgements)

責任ある開示（Responsible Disclosure）にご協力いただいた全てのセキュリティ研究者の方々に深く感謝申し上げます。
ご要望に応じて、脆弱性の修正リリース時に、**皆様のお名前をクレジット**させていただきます。
